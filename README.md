# TemichevVet CRM

Production-репозиторий CRM для ветеринарной клиники.

Сейчас в репозитории есть рабочий прототип интерфейса по дереву и процессам Vet.AF, а также документы, фиксирующие MVP и выбранный стек.

- [Production Stack](docs/STACK.md) — выбранный технический стек.

## Что уже заложено

- Живая очередь: первичный клиент или существующий владелец/пациент, срочность, сотрудник без обязательного выбора врача, кабинет, комментарий.
- Расписание: календарная лента дат, запись на приём, зависимость от графика сотрудников.
- Владельцы: профиль, организация, телефоны, адрес, email, источник, паспортные данные, скидки, личный кабинет, чат, SMS, Telegram, Push, пациенты, приёмы, баланс, депозит, доверенные лица.
- Пациенты: профиль животного, обязательные выпадающие списки вида и породы, пол, дата рождения, окрас, микрочип, клеймо, стерилизация, избранное, статус, вес, вакцинации, задачи и история приёмов.
- Приём: лист осмотра, анамнез, осмотр, симптомы, вес, температура, диагнозы, манипуляции, рекомендации, план лечения, товары и услуги, документы, стационар.
- Документы: текстовые шаблоны, составные шаблоны, уведомления, переменные, выбор шаблона внутри приёма.
- Склад: товары, услуги, расходные товары в услуге, поставки, накладные, импорт файла поставки, сроки годности, стеллажи, полки, серии, печать ценников.
- Финансы: счета, платежи, тип оплаты, статусы оплаты, товарная накладная, продажи без приёма, депозит владельца.
- Стационар: боксы, пациенты, длительность пребывания, дневные назначения.
- Настройки: организация, реквизиты, тарифы и платежи, история операций, филиал, режим работы, ночной режим, графики, события, расходы, зарплатные профили, телефония, лаборатории, стационар, кабинеты, онлайн-кассы, склады.
- Сотрудники: профиль, статус, должность, раздел по умолчанию, длительность приёма, доступы по модулям, склады, уведомления, события.
- AI-помощник оставлен как отдельный будущий модуль.

## Безопасность аудита

В живой CRM были просмотрены страницы, формы и вкладки без сохранения рабочих изменений. Исключения: ранее открытие страницы создания продажи в Vet.AF создало пустую продажу на 0 ₽; при повторной проверке прямой переход на создание приёма открыл черновик приёма. Эти записи не редактировались, не оплачивались и не удалялись.

## Запуск

Backend-инфраструктура:

```bash
docker compose up -d postgres redis minio
docker compose up -d api
```

API:

- `http://127.0.0.1:4000/api/health`
- `http://127.0.0.1:4000/api/docs`

PostgreSQL доступен с хоста на `127.0.0.1:5433`.

Локальный директор для разработки создаётся seed-скриптом:

- логин: `+70000000001`
- пароль: `ChangeMe123!`

В production пароль директора должен задаваться через переменные окружения и сразу заменяться.

Frontend CRM:

```bash
npm run web:dev
```

По умолчанию интерфейс доступен на `http://127.0.0.1:3000`.

Production-сборка frontend:

```bash
npm run web:build
```

Локальный production-режим через Docker:

```bash
docker compose build api web
docker compose up -d postgres redis minio api web
```

Web-контейнер доступен на `http://127.0.0.1:3000`, отдаёт собранный React frontend через Nginx и проксирует `/api` к backend. Для доступа с рабочих мест в локальной сети задайте `WEB_BIND_ADDR=0.0.0.0` и `APP_URL=http://IP_СЕРВЕРА:3000`.

Быстрый запуск на новом компьютере клиники:

```bash
npm run clinic:start
```

На macOS можно запустить двойным кликом файл `start-temichevvet.command` в корне проекта. Он поднимет локальный сервер и откроет CRM в браузере.

На Windows можно запустить двойным кликом файл `start-temichevvet-windows.bat` в корне проекта. Он вызывает PowerShell-скрипт `scripts/start-clinic-server.ps1`, поднимает локальный сервер и открывает CRM в браузере.

Если код обновлялся или CRM перенесена на новый компьютер и web/api нужно пересобрать:

```bash
npm run clinic:start:build
```

Скрипт создаст `.env` при первом запуске, поднимет PostgreSQL, Redis, MinIO, backend и web через Docker Compose, а затем покажет адрес для этого компьютера и адрес для других компьютеров в локальной сети. Чтобы сразу открыть CRM в браузере на серверном компьютере:

```bash
npm run clinic:start:open
```

### Ручное обновление Docker-образов из GitHub

По умолчанию локальная установка запускается от образов, загруженных с флешки:

```dotenv
TEMICHEVVET_API_IMAGE=temichevvet-api:local
TEMICHEVVET_WEB_IMAGE=temichevvet-web:local
```

Для интернет-обновлений используются Docker-образы из GitHub Container Registry:

```dotenv
TEMICHEVVET_REMOTE_API_IMAGE=ghcr.io/pivotemnoe/kliniksrm-api:stable
TEMICHEVVET_REMOTE_WEB_IMAGE=ghcr.io/pivotemnoe/kliniksrm-web:stable
TEMICHEVVET_AUTO_PULL_IMAGES=true
```

Обычный запуск CRM проверяет свежие Docker-образы программы и обновляет `api`/`web`, если интернет и реестр доступны. Перед интернет-обновлением создаётся backup базы в `backups/`. Данные клиники хранятся в Docker volumes и не удаляются.

На Windows также есть ручная кнопка `Обновить TemichevVet через интернет`: она принудительно создаёт backup базы, скачивает свежие `api` и `web` образы и перезапускает контейнеры.

Проверить установленную сборку можно кнопкой `Проверить версию TemichevVet` на серверном Windows-компьютере или в CRM: `Настройки` -> `Система и backup`. Новые Docker-образы показывают commit, дату сборки и источник образа.

Для ручного обновления из терминала:

```bash
npm run clinic:start:update
```

На Windows аналогичная команда:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\start-clinic-server.ps1 -UpdateImages -Open
```

GitHub Actions публикует образы при push в `main`. Если пакеты GHCR приватные, на серверном компьютере один раз выполните `docker login ghcr.io`.

Чтобы заранее скачать официальный установщик Docker Desktop для Windows в папку `installers`:

```bash
npm run clinic:download-docker-windows
```

На Windows то же самое можно выполнить так:

```powershell
powershell -ExecutionPolicy Bypass -File scripts\download-docker-windows-installer.ps1
```

Для установки на новый Windows-компьютер без подготовки: установите Docker Desktop из файла `installers\Docker Desktop Installer.exe`, перезагрузите компьютер при необходимости, откройте Docker Desktop, затем запустите `start-temichevvet-windows.bat`.

## Флешка-установщик

Можно собрать переносной комплект на флешку. Это не BIOS-загрузочная флешка с операционной системой, а флешка-установщик CRM: на ней будут отдельные кнопки для Windows, macOS и Linux.

На macOS сначала подключите флешку и посмотрите её имя в Finder. Обычно путь будет вида `/Volumes/FLASH_NAME`.

Формат флешки лучше выбрать `exFAT`: её прочитают Windows, macOS и Linux, а файлы больше 4 ГБ будут поддерживаться. `FAT32` не подходит для комплекта с Docker-образами.

Быстрая сборка комплекта:

```bash
npm run clinic:make-flash -- /Volumes/FLASH_NAME
```

Сборка комплекта с Docker-образами для установки почти без интернета:

```bash
npm run clinic:make-flash -- --include-images /Volumes/FLASH_NAME
```

На флешке появится папка `TemichevVet-Portable`:

- `Установить TemichevVet - Windows.bat`
- `Установить TemichevVet - Mac.command`
- `Установить TemichevVet - Linux.sh`
- `README.txt`
- `CRM/`
- `installers/`
- `docker-images/`, если использован режим `--include-images`.

Установщик копирует CRM в домашнюю папку пользователя `TemichevVet`, не перезаписывает локальный `.env`, загружает Docker-образы из комплекта при наличии и запускает локальный сервер.

Данные медицинских карточек, очереди, записей, приёмов, склада и стационара идут через backend API и сохраняются в PostgreSQL. Токен авторизации в `localStorage` не хранится: используется серверная HttpOnly cookie-session.

## Требования к локальному серверу клиники

Минимум для тестов:

- Windows 10/11 64-bit, macOS или Linux.
- 8 ГБ оперативной памяти.
- SSD от 128 ГБ.
- Docker Desktop или Docker Engine.

Рекомендуется для клиники:

- Windows 11 Pro 64-bit, macOS на Apple Silicon/Intel или Linux.
- 16 ГБ оперативной памяти.
- SSD/NVMe от 256 ГБ, лучше 512 ГБ.
- Стабильная локальная сеть.
- Бесперебойник.
- Внешний диск или NAS для ежедневных backup.

Для Windows нужен включённый WSL 2. Docker Desktop обычно помогает включить нужные компоненты, но иногда Windows попросит перезагрузку.
