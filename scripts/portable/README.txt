TemichevVet Portable

Это переносной комплект для установки TemichevVet CRM с флешки.

Состав папки:
- CRM/ - чистая копия приложения без рабочей базы и без node_modules.
- installers/ - установщики, если они были добавлены в комплект.
- docker-images/ - готовые Docker-образы, если комплект собран с режимом --include-images.
- portable/ - служебные скрипты установки.
- VERSION.txt - дата сборки комплекта и commit.

Как запускать:

Windows:
Серверный компьютер, где будет храниться база:
1. Если Docker Desktop не установлен, запустите installers/Docker Desktop Installer.exe.
2. После установки перезагрузите компьютер, если Windows попросит.
3. Откройте Docker Desktop и дождитесь, чтобы он запустился.
4. Запустите "Установить TemichevVet - Windows.bat".
5. После установки на рабочем столе и в меню "Пуск" появится ярлык "TemichevVet" с логотипом клиники.
   Дальше запускайте CRM через этот ярлык.

Проверка установленной версии Windows:
- Запустите "Проверить версию TemichevVet - Windows.bat" с флешки или ярлык "Проверить версию TemichevVet" на сервере.
- В самой CRM откройте "Настройки" -> "Система и backup": там показаны версия API, commit и дата сборки.

Обновление уже установленного серверного компьютера Windows:
1. Вставьте флешку со свежим комплектом.
2. Запустите "Обновить TemichevVet - Windows.bat".
3. Скрипт попробует создать резервную копию базы в папке:
   %USERPROFILE%\TemichevVet\backups
4. Затем программа обновится, Docker-образы загрузятся с флешки, контейнеры пересоздадутся.
5. Данные клиники не удаляются: владельцы, пациенты, приёмы, счета и файлы остаются в Docker volumes.

Важно: обновлятор не удаляет Docker volumes и не перезаписывает файл .env. Поэтому пароли, порты и локальные данные установленной клиники сохраняются.

Ручное обновление серверного компьютера Windows через интернет:
1. Один раз обновите сервер с флешки, чтобы появились новые кнопки обновления.
2. Если адреса образов ещё не настроены, запустите "Настроить обновления GitHub - Windows.bat".
   По умолчанию используются образы:
   ghcr.io/pivotemnoe/kliniksrm-api:stable
   ghcr.io/pivotemnoe/kliniksrm-web:stable
3. Для установки новой версии запустите "Обновить TemichevVet через интернет - Windows.bat".
4. Скрипт сначала создаст резервную копию базы в:
   %USERPROFILE%\TemichevVet\backups
5. Затем скачает свежие Docker-образы из GitHub Container Registry, перезапустит CRM и применит миграции базы.
6. После обновления запустите "Проверить версию TemichevVet" или откройте "Система и backup" в CRM.

Обычный ярлык "TemichevVet" тоже проверяет обновление программы при запуске, если интернет и GitHub Container Registry доступны.
Перед таким обновлением создаётся резервная копия базы в:
   %USERPROFILE%\TemichevVet\backups
Ручная кнопка остаётся для случая, когда нужно принудительно проверить и установить свежую версию.

Дополнительный компьютер врача или администратора:
1. Убедитесь, что серверный компьютер включён, не спит и TemichevVet на нём запущен.
2. Подключите дополнительный компьютер к той же Wi-Fi/локальной сети.
3. Запустите "Подключить рабочее место - Windows.bat".
4. Установщик найдёт сервер в сети, сохранит адрес и создаст ярлык "TemichevVet" на рабочем столе.
5. На дополнительный компьютер Docker ставить не нужно.

Если автоматический поиск не нашёл сервер, установщик попросит ввести адрес вручную.
На серверном компьютере при запуске CRM показывается строка "Open from other clinic computers".
Вводить нужно именно этот адрес, например:

  http://192.168.1.15:3000

Адрес http://127.0.0.1:3000 подходит только самому серверному компьютеру.
На втором компьютере он означает "этот второй компьютер", поэтому к серверу не подключит.

Если нужно заранее записать адрес на флешку, можно рядом с кнопками создать файл server-url.txt
и написать в него адрес серверного компьютера.

Если при запуске появляется ошибка вида:
- failed to resolve source metadata for docker.io/library/node;
- TLS handshake timeout;
- failed to request registry-1.docker.io.

Это значит, что Docker работает, но компьютер не смог скачать базовые образы из Docker Hub.
Решение: пересоздайте флешку на рабочем компьютере с готовыми образами:

  scripts/create-portable-flash.sh --include-images --platform linux/amd64 /Volumes/FLASH

После этого на Windows снова запустите "Установить TemichevVet - Windows.bat".
Установщик загрузит docker-images/temichevvet-images.tar с флешки и запустит CRM без пересборки api/web.
Если CRM уже установлена и нужно только обновить программу, запустите "Обновить TemichevVet - Windows.bat".

Если появляется ошибка вида:
- pull access denied for temichevvet-web;
- pull access denied for temichevvet-api;
- No such image: temichevvet-api:local;
- No such image: temichevvet-web:local.

Это значит, что готовые локальные Docker-образы не загрузились с флешки или архив docker-images/temichevvet-images.tar повреждён.
Решение: пересоздайте флешку с режимом --include-images и после сборки убедитесь, что файл docker-images/temichevvet-images.tar есть на флешке.

Если Docker показывает ошибку про порт 9000 или 9001:
- bind for 127.0.0.1:9000 failed;
- port is already allocated.

Это значит, что на компьютере уже занята локальная служба MinIO. Установщик сам попробует выбрать свободные порты, например 9100 и 9101. Если ошибка осталась, закройте старые контейнеры в Docker Desktop или удалите старый проект TemichevVet без удаления volumes с данными.

Если Docker показывает "Virtualization support not detected":
1. Перезагрузите компьютер.
2. Зайдите в BIOS/UEFI клавишей F2 или Del сразу после включения.
3. Найдите Intel Virtualization Technology, VT-x или Virtualization.
4. Поставьте Enabled.
5. Сохраните настройки через F10 и загрузите Windows.
6. Откройте Docker Desktop ещё раз.

Если после этого Docker всё равно не запускается, включите в Windows компоненты:
- Windows Subsystem for Linux;
- Virtual Machine Platform;
- Hyper-V, если он доступен в вашей редакции Windows.

macOS:
1. Установите Docker Desktop для macOS, если он ещё не установлен.
2. Откройте Docker Desktop и дождитесь, чтобы он запустился.
3. Запустите "Установить TemichevVet - Mac.command".
4. Для обновления уже установленной CRM запустите "Обновить TemichevVet - Mac.command".

Linux:
1. Установите Docker Engine и Docker Compose Plugin.
2. Запустите "Установить TemichevVet - Linux.sh".
3. Для обновления уже установленной CRM запустите "Обновить TemichevVet - Linux.sh".

Что делает установщик:
- копирует CRM в домашнюю папку пользователя: TemichevVet;
- не перезаписывает локальный файл .env, если он уже создан;
- загружает Docker-образы из docker-images/temichevvet-images.tar, если такой файл есть;
- при обновлении пытается сделать backup базы перед копированием новой версии;
- создаёт ярлык "TemichevVet" на рабочем столе и в меню "Пуск";
- запускает локальный сервер CRM;
- открывает адрес http://127.0.0.1:3000/login на этом компьютере.

Что делает подключение рабочего места:
- не ставит Docker;
- не копирует серверную базу;
- сначала проверяет сохранённый адрес сервера;
- проверяет стандартные имена temichevvet.local и temichevvet;
- проверяет уже видимые компьютеры в сети;
- затем ищет TemichevVet в основной Wi-Fi/локальной сети по портам 3000-3005;
- если сервер не найден, просит ввести адрес вручную;
- проверяет найденный адрес через /api/health;
- сохраняет адрес в папке TemichevVet-Workstation;
- создаёт ярлык "TemichevVet" на рабочем столе и в меню "Пуск".

Обновление через интернет:
- если в файле .env настроены TEMICHEVVET_REMOTE_API_IMAGE и TEMICHEVVET_REMOTE_WEB_IMAGE,
  обычный запуск и кнопка интернет-обновления подтянут свежие api/web Docker-образы из реестра;
- если GitHub Container Registry пакеты приватные, один раз выполните на сервере:
  docker login ghcr.io
- автообновление при обычном запуске включено:
  TEMICHEVVET_AUTO_PULL_IMAGES=true.
- обновляются только Docker-образы программы; Docker volumes с данными клиники не удаляются.

Важно:
- Это не BIOS-загрузочная флешка с операционной системой.
- Это флешка-установщик приложения.
- Рабочие данные клиники хранятся в Docker volumes на компьютере, куда установлена CRM.
- Для чистой установки не переносите backup рабочей базы, если не хотите переносить реальные записи.
