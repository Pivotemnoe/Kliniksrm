# Развёртывание owner-gateway

Комплект предназначен только для российского сервера `193.188.23.65`.
Нидерландский узел `5.129.239.104` используется исключительно как SSH-мост и не хранит файлы, контейнеры или данные кабинета.

## Что хранится на сервере

- Docker-образ `temichevvet-owner-gateway` с публичным интерфейсом;
- отдельный PostgreSQL volume `temichevvet_owner_gateway_postgres_data`;
- ежедневные SQL-бэкапы в `./backups` с хранением 14 дней;
- `.env` с отдельными секретами шлюза и токеном MAX.
- VAPID-ключи для push-уведомлений установленного личного кабинета.

Шлюз не подключается к базе CRM. Клиника передаёт ему только разрешённые снимки владельцев исходящими HTTPS-запросами.

## Безопасный порядок первого запуска

1. Загрузить AMD64-образ и эти файлы прямо на российский сервер.
2. Положить токен MAX в временный файл `.max-bot-token` с правами `600`.
3. Запустить `create-server-env.sh`, затем удалить `.max-bot-token`.
4. Проверить `docker-compose config`.
5. Запустить только `postgres`, дождаться health-check и проверить наличие таблицы `OwnerSnapshot`.
6. Запустить `gateway` и `backup`, проверить `http://127.0.0.1:4100/health`.
7. Подключить временную HTTP-конфигурацию Nginx и проверить `nginx -t` до reload.
8. Создать DNS `cabinet.temichevvet.ru -> 193.188.23.65`.
9. Выпустить сертификат, заменить конфигурацию на HTTPS и снова выполнить `nginx -t` до reload.
10. Только после внешней HTTPS-проверки зарегистрировать MAX webhook.

Docker volumes этого комплекта никогда не удаляются при обновлении образа или конфигурации.
